# Code Review Workflow

When this workflow is triggered:

1. Review code structure.
2. Identify bugs.
3. Check readability.
4. Check maintainability.
5. Check performance concerns.
6. Check security issues.
7. Check scalability concerns.
8. Suggest improvements.
9. Explain reasoning briefly.
10. Avoid unnecessary refactoring.

Response Format:

STRENGTHS:
<good practices>

ISSUES:
<problems found>

IMPROVEMENTS:
<recommended changes>

PRIORITY:
High / Medium / Low
