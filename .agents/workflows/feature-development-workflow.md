# Feature Development Workflow

When this workflow is triggered:

1. Understand the feature request.
2. Identify affected files.
3. Explain the implementation approach.
4. Follow existing project structure.
5. Generate production-ready code.
6. Use Node.js, Express, MongoDB, Mongoose, EJS and Bootstrap best practices.
7. Follow REST API conventions.
8. Use async/await whenever possible.
9. Preserve existing functionality.
10. Provide integration instructions.

Response Format:

FILE:
<file path>

PURPOSE:
<why the change is needed>

CODE:
<code>

INTEGRATION:
<where to place the code>

NOTES:
<any warnings or side effects>
