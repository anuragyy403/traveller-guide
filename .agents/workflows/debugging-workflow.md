# Debugging Workflow

When this workflow is triggered:

1. Read the error carefully.
2. Identify the exact root cause.
3. Explain why the issue occurs.
4. Identify affected files.
5. Suggest the smallest possible fix.
6. Preserve existing functionality.
7. Avoid rewriting unrelated code.
8. Mention possible side effects.
9. Explain how to verify the fix.
10. Recommend best practices if relevant.

Response Format:

ERROR:
<error summary>

ROOT CAUSE:
<why it happens>

FILE:
<affected file>

FIX:
<code>

VERIFICATION:
<how to test>
