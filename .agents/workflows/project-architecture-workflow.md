# Project Architecture Workflow

When this workflow is triggered:

1. Analyze application requirements.
2. Separate routes, controllers, models, and views.
3. Maintain clean folder structure.
4. Follow REST API conventions.
5. Recommend reusable solutions.
6. Avoid unnecessary complexity.
7. Consider scalability.
8. Consider maintainability.
9. Explain architecture decisions.
